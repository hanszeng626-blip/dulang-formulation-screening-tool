# 独狼制剂处方筛选综合工具

这是一个面向化学药品制剂研发早期处方设计与筛选的静态网页原型。当前版本先把框架搭好，用于录入原料药性质、原研/参比信息、初始处方、检验结果，并形成初步评价建议和汇报摘要。

## 当前能力

- 原料药性质录入：名称、化学式、剂量、密度、流动性、溶解性、pKa、BCS、吸湿性和风险备注。
- 制剂项目录入：制剂名称、剂型、给药途径、研发类型、参比/原研信息和 QTPP。
- 原研/参比检索：根据药品名生成 Google、DailyMed、Drugs@FDA、Orange Book、openFDA 等检索入口，并支持从粘贴资料中提取剂型、规格、申请号、申请人、辅料线索等字段。
- 处方筛选：固定初始处方框，支持录入处方型号、处方依据、设计说明、处方成分/型号和处方比例；默认包含主药、填充剂、崩解剂、功能辅料、润滑剂、其他辅料，并可增减处方行。
- 检验结果：片重、硬度、脆碎度、崩解、含量均匀度、溶出和含量。
- 评价输出：处方组成图、关键指标评价、下一步建议、研发总结草稿。
- 本地保存与导出：草稿保存到浏览器 localStorage；可导出 JSON 和 txt。

## 本地运行

这是纯静态页面，直接打开 `index.html` 即可；也可以启动一个静态服务器：

```powershell
python -m http.server 5174
```

然后访问 `http://localhost:5174`。

## Render 部署

本仓库已包含 `render.yaml`，可在 Render 中作为 Static Site 部署。

- Build Command: 留空
- Publish Directory: `.`

## 后续规划

1. 建立结构化数据库：API 性质、辅料库、参比信息、处方版本、工艺参数、检验结果。
2. 扩展剂型模板：片剂、胶囊、颗粒、口服液、注射剂分别建立 CQA 和评价规则。
3. 引入 DOE：支持单因素、正交、Plackett-Burman、响应面和混料设计。
4. 连接公开数据源：FDA IID、Orange Book、DailyMed、药典标准、企业内部原始记录。
5. 增加报告输出：生成研发总结、处方筛选报告、工艺开发概览和注册资料草稿。
6. 增加后端联网模块：通过 Google Programmable Search、openFDA、DailyMed API 等合规接口自动检索并结构化导入原研/参比信息。
